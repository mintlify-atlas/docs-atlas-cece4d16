Monthly reports provide detailed analytics on access failures, blocked users, and statistics aggregated by faculty, department, and contractor companies over a calendar month period.

## Report Period Selection

Monthly reports use the **Mensual** period type, which calculates the full calendar month:

```javascript
// From ReportesView.jsx:50-60
if (tipo === 'mensual') {
  const inicio = new Date(ahora.getFullYear(), ahora.getMonth() - offset, 1);
  inicio.setHours(0, 0, 0, 0);
  const finMes = new Date(inicio.getFullYear(), inicio.getMonth() + 1, 0, 23, 59, 59, 999);
  const hasta  = offset === 0 ? new Date(ahora) : finMes;
  return {
    desde:    inicio,
    hasta,
    etiqueta: inicio.toLocaleDateString('es-CO', { month: 'long', year: 'numeric' }),
  };
}
```

### Period Calculation

- **Current month** (offset = 0): From 1st of month at 00:00:00 to current date/time
- **Previous month** (offset = 1): From 1st to last day (23:59:59) of previous month
- **Label format**: "enero 2026", "febrero 2026", etc. (Spanish locale)

### Navigation

Use the arrow buttons to navigate between months:

- **← Previous**: View older month (offset + 1)
- **→ Next**: View newer month (offset - 1)
- Current month indicator: Arrow button disabled when offset = 0

## Failures by Month

The primary metric in monthly reports is the total access failures recorded during the period.

### Total Failures

Displayed in the summary card:

```jsx
// From ReportesView.jsx:347-354
<ResumenCard
  titulo="Fallas en el período"
  valor={reporte?.fallasPeriodo}
  subtitulo="Reportes sin carnet"
  icono="⚠️"
  color={{ bg: 'bg-yellow-50', text: 'text-yellow-500' }}
/>
```

**Interpretation:**
- Each failure represents one instance of a user entering without their ID card
- Failures can be due to: forgotten, lost, or stolen cards
- High failure counts may indicate need for awareness campaigns

### Failures by User Type

Monthly reports break down failures by user role:

- **Student failures** (`fallasEstudiantes`): Students who forgot/lost cards
- **Employee failures** (`fallasEmpleados`): Staff access failures
- **Contractor failures** (`fallasContratistas`): External contractor failures

**Export data includes:**
```javascript
// From ReportesView.jsx:179-181
['Total fallas', 'Fallas de estudiantes', 'Fallas de empleados', 'Fallas de contratistas'],
[reporte.fallasPeriodo ?? 0, reporte.fallasEstudiantes ?? 0, 
 reporte.fallasEmpleados ?? 0, reporte.fallasContratistas ?? 0]
```

### Daily Breakdown

View failures distributed across each day of the month:

```javascript
// Data structure: reporte.fallasPorDia
[
  { dia: "2024-01-15", total: 12 },
  { dia: "2024-01-16", total: 8 },
  { dia: "2024-01-17", total: 15 }
]
```

**Visualization:**
- Horizontal bar chart using Recharts library (ReportesView.jsx:365-400)
- Each day shown as a separate bar
- Color-coded using GANTT_COLORS palette
- Interactive tooltips show exact counts
- Chart height adapts to number of days: `Math.max(fallasPorDia.length * 42, 120)`

## Blocked Users Report

While not directly shown in the main report view, blocked users data can be analyzed through:

### Blocked User Indicators

Users become blocked when:

1. **4th failure due to forgotten card** → Automatic block
2. **Card reported lost** → Immediate block
3. **Card reported stolen** → Immediate block
4. **Administrative action** → Manual block

See the [Block Management](/admin/blocking-management) documentation for details on blocked users.

### Monthly Block Statistics

To analyze blocks in a monthly context:

- Query `Historial_Bloqueos` table for blocks in date range
- Filter by `fecha_bloqueo` within month period
- Group by `motivo_bloqueo` (limit_fallas, perdida_carnet, robo_carnet)

## Exporting Monthly Reports

Click the **"Descargar Excel"** button to export the current month's report:

```jsx
// From ReportesView.jsx:310-319
<button
  onClick={exportarExcel}
  disabled={!reporte || loading || exportando}
  className="flex items-center gap-2 text-sm font-medium px-4 py-2 rounded-lg border border-green-700 text-green-700 hover:bg-green-50 transition-colors disabled:opacity-40 disabled:cursor-not-allowed"
>
  {exportando ? 'Exportando...' : 'Descargar Excel'}
</button>
```

**Generated filename:**
```javascript
const fecha = new Date().toISOString().slice(0, 10);
XLSX.writeFile(wb, `informe_ucc_${periodo}_${fecha}.xlsx`);
// Example: informe_ucc_mensual_2026-03-13.xlsx
```

See [Excel Export](/admin/reports/export-excel) for complete details on the export format.

## Best Practices

### Monthly Review Workflow

1. **At month end**: Generate complete month report (offset = 1)
2. **Export to Excel**: Download for record-keeping
3. **Analyze trends**: Compare with previous months
4. **Identify issues**: High-failure programs/departments
5. **Take action**: Awareness campaigns, policy updates

### Key Metrics to Monitor

- **Failure rate**: (Total failures / Total users) × 100
- **Repeat offenders**: Users with 3+ failures (at risk of blocking)
- **Block rate**: New blocks this month
- **Top programs**: Programs with disproportionate failures

### Recommendations

- Review monthly reports at administrative meetings
- Track failure trends over multiple months
- Set target failure rates by user type
- Implement interventions for high-failure groups
- Archive Excel exports for historical analysis
