The Excel export functionality allows administrators to download comprehensive reports in `.xlsx` format for offline analysis, archival, and sharing with stakeholders.

## Export Process

The export process is triggered from the Reports view with a single button click.

### User Interface

**Export button:**
```jsx
// From ReportesView.jsx:310-319
<button
  onClick={exportarExcel}
  disabled={!reporte || loading || exportando}
  className="flex items-center gap-2 text-sm font-medium px-4 py-2 rounded-lg border border-green-700 text-green-700 hover:bg-green-50 transition-colors disabled:opacity-40 disabled:cursor-not-allowed"
>
  <svg xmlns="http://www.w3.org/2000/svg" fill="none" viewBox="0 0 24 24" strokeWidth={1.8} stroke="currentColor" className="w-4 h-4">
    <path strokeLinecap="round" strokeLinejoin="round" d="M3 16.5v2.25A2.25 2.25 0 0 0 5.25 21h13.5A2.25 2.25 0 0 0 21 18.75V16.5M16.5 12 12 16.5m0 0L7.5 12m4.5 4.5V3" />
  </svg>
  {exportando ? 'Exportando...' : 'Descargar Excel'}
</button>
```

**Button states:**
- **Active**: Green border, hover effect, download icon visible
- **Disabled**: Grayed out when no data, loading, or export in progress
- **Exporting**: Shows "Exportando..." text during file generation

### Export Function

The main export function (ReportesView.jsx:162-256):

```javascript
const exportarExcel = () => {
  if (!reporte) return;
  setExportando(true);

  try {
    const wb = XLSX.utils.book_new();
    // ... create sheets ...
    const fecha = new Date().toISOString().slice(0, 10);
    XLSX.writeFile(wb, `informe_ucc_${periodo}_${fecha}.xlsx`);
  } finally {
    setExportando(false);
  }
};
```

**Process flow:**
1. Check if report data exists
2. Set loading state (`exportando = true`)
3. Create new workbook
4. Generate and append sheets
5. Write file to disk with formatted filename
6. Reset loading state

The generated Excel workbook contains up to **7 sheets**, depending on available data.

## Data Included in Exports

The export includes all available data from the current report view.

### Required Data (Always Included)

**Resumen sheet contains:**
- Report title
- Period label (e.g., "marzo 2026")
- Generation timestamp
- Total registered users by type (students, employees, contractors)
- Total failures by type

### Optional Data (Conditionally Included)

Sheets are only created if the corresponding data exists:

```javascript
if (reporte.fallasPorDia?.length) { /* create sheet */ }
if (reporte.porPrograma?.length) { /* create sheet */ }
if (reporte.porDependencia?.length) { /* create sheet */ }
if (reporte.fallasPorPrograma?.length) { /* create sheet */ }
if (reporte.fallasPorDependencia?.length) { /* create sheet */ }
if (reporte.fallasPorEmpresa?.length) { /* create sheet */ }
```

**Result:**
- Minimum sheets: 1 (Resumen only)
- Maximum sheets: 7 (all data available)

### Data Transformations

**Array-to-sheet conversion:**
The Resumen sheet uses `XLSX.utils.aoa_to_sheet()` for array-of-arrays format.

**JSON-to-sheet conversion:**
All other sheets use `XLSX.utils.json_to_sheet()` for easier data mapping:

```javascript
// Map data objects to Excel-friendly format
reporte.fallasPorDia.map(f => ({ 
  Día: f.dia,              // Column header
  'Total fallas': f.total  // Column header with value
}))
```

## Using the xlsx Library

The system uses the [SheetJS xlsx](https://www.npmjs.com/package/xlsx) library for Excel file generation.

### Installation

From package.json:13:
```json
"xlsx": "^0.18.5"
```

Install via npm:
```bash
npm install xlsx
```

### Import

```javascript
// From ReportesView.jsx:6
import * as XLSX from 'xlsx';
```

### Core Functions Used

#### 1. Create New Workbook

```javascript
const wb = XLSX.utils.book_new();
```

Creates an empty workbook object to which sheets can be added.

#### 2. Array-of-Arrays to Sheet

```javascript
const ws = XLSX.utils.aoa_to_sheet([
  ['Header 1', 'Header 2'],
  ['Value 1', 'Value 2']
]);
```

Converts a 2D array to a worksheet. Used for the Resumen sheet with manual layout.

#### 3. JSON to Sheet

```javascript
const ws = XLSX.utils.json_to_sheet([
  { Programa: 'Ingeniería', Estudiantes: 245 },
  { Programa: 'Derecho', Estudiantes: 189 }
]);
```

Converts an array of objects to a worksheet. Keys become column headers.

#### 4. Set Column Widths

```javascript
ws['!cols'] = [{ wch: 40 }, { wch: 15 }];
```

Sets column widths in characters. Array index corresponds to column (0 = A, 1 = B, etc.).

#### 5. Append Sheet to Workbook

```javascript
XLSX.utils.book_append_sheet(wb, ws, 'Sheet Name');
```

Adds a worksheet to the workbook with the specified tab name.

#### 6. Write File to Disk

```javascript
XLSX.writeFile(wb, 'filename.xlsx');
```

Triggers browser download of the Excel file with the specified name.

### File Naming Convention

```javascript
// From ReportesView.jsx:251-252
const fecha = new Date().toISOString().slice(0, 10);
XLSX.writeFile(wb, `informe_ucc_${periodo}_${fecha}.xlsx`);
```

**Format:** `informe_ucc_{periodo}_{fecha}.xlsx`

**Examples:**
- Daily report: `informe_ucc_diario_2026-03-13.xlsx`
- Weekly report: `informe_ucc_semanal_2026-03-13.xlsx`
- Monthly report: `informe_ucc_mensual_2026-03-13.xlsx`
- Semester report: `informe_ucc_semestral_2026-03-13.xlsx`

**Components:**
- `informe_ucc`: Fixed prefix
- `{periodo}`: diario, semanal, mensual, or semestral
- `{fecha}`: ISO date (YYYY-MM-DD) when file was generated
- `.xlsx`: Excel file extension

## Performance Considerations

### File Generation Time

Export time depends on data volume:

- **Small reports** (less than 100 rows): Instant (less than 100ms)
- **Medium reports** (100-1000 rows): less than 500ms
- **Large reports** (more than 1000 rows): 500ms-2s

### Memory Usage

Excel generation happens in-memory:

- Workbook and all sheets stored in JavaScript memory
- File written to disk in one operation
- Memory freed after download completes

**Typical memory usage:**
- 7 sheets with 500 total rows: ~2-3 MB
- 7 sheets with 5000 total rows: ~10-15 MB

### Optimization Tips

1. **Conditional sheet creation**: Only create sheets with data
2. **Minimal data transformation**: Direct property mapping
3. **Fixed column widths**: Pre-calculated, not dynamic
4. **No formatting**: Plain data for faster generation

## Troubleshooting

### Export Button Disabled

**Possible causes:**
- No report data loaded yet
- Report still loading (`loading = true`)
- Export already in progress (`exportando = true`)

**Solution:** Wait for data to load, then retry

### Missing Sheets

**Cause:** No data for that category

**Example:**
- If no contractors have failures, "Fallas por empresa" sheet not created
- If `fallasPorDia` array is empty, "Fallas por día" sheet omitted

**Solution:** This is expected behavior, not an error

### File Not Downloading

**Possible causes:**
- Browser blocking pop-ups/downloads
- Insufficient browser storage
- Browser compatibility issue

**Solutions:**
- Allow downloads for the site
- Clear browser cache/storage
- Try different browser
