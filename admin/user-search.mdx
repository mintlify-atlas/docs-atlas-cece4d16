The User Search functionality (implemented in `EstudiantesView.jsx`) allows administrators to search, filter, and manage all users in the system with detailed profile information.

## Search Interface

### Search Bar

The search bar supports multiple search criteria:

```jsx
<input
  type="text"
  placeholder="Buscar por nombre, ID, rol o información..."
  value={busqueda}
  onChange={e => setBusqueda(e.target.value)}
/>
```

**Searchable Fields:**
- `nombre_completo` - Full name
- `id_institucional` - Institutional ID (e.g., 80123456)
- `infoExtra` - Additional role-specific information
- `roles` - User roles (Estudiante, Empleado, Contratista)

<Tip>
  The search is case-insensitive and performs real-time filtering as you type.
</Tip>

### Filter Buttons

Quick filters for access status:

```jsx
<div className="flex gap-1 bg-white border border-gray-200 rounded-xl">
  {['todos', 'activo', 'bloqueado'].map(op => (
    <button
      onClick={() => setFiltroAcceso(op)}
      className={filtroAcceso === op ? 'bg-green-700 text-white' : 'text-gray-500'}
    >
      {op === 'todos' ? 'Todos' : op === 'activo' ? 'Activos' : 'Bloqueados'}
    </button>
  ))}
</div>
```

## Filtering Logic

The filtering combines search and status filters:

```javascript
const filtrados = usuarios.filter(u => {
  // Search filter
  const coincideBusqueda =
    u.nombre_completo?.toLowerCase().includes(busqueda.toLowerCase()) ||
    u.id_institucional?.toLowerCase().includes(busqueda.toLowerCase()) ||
    u.infoExtra?.toLowerCase().includes(busqueda.toLowerCase()) ||
    u.roles?.some(r => r.toLowerCase().includes(busqueda.toLowerCase()));

  // Status filter
  const coincideAcceso =
    filtroAcceso === 'todos' || u.acceso === filtroAcceso;

  return coincideBusqueda && coincideAcceso;
});
```

## User Data Structure

The user object returned from `getUsuarios()` has this structure:

```typescript
interface Usuario {
  id_institucional: string;        // e.g., '80123456'
  nombre_completo: string;         // Full name
  documento_identidad: string;     // National ID
  correo_institucional: string;    // @ucc.edu.co email
  roles: string[];                 // ['Estudiante', 'Empleado', ...]
  infoExtra?: string;             // Concatenated role info
  infoItems?: Array<{             // Structured role info
    tipo: 'programa' | 'dependencia' | 'empresa';
    valor: string;
  }>;
  total_fallas: number;           // 0-4
  acceso: 'activo' | 'bloqueado'; // Current status
}
```

## Database Queries

The user list comes from the `info_estudiantes`, `info_empleados`, and `info_contratistas` tables joined with `usuarios`:

```sql
SELECT 
  u.id_institucional,
  u.nombre_completo,
  u.total_fallas,
  u.acceso,
  -- Role-specific info from joined tables
FROM usuarios u
LEFT JOIN info_estudiantes ie ON u.id_institucional = ie.id_institucional
LEFT JOIN info_empleados em ON u.id_institucional = em.id_institucional
LEFT JOIN info_contratistas ct ON u.id_institucional = ct.id_institucional
```
